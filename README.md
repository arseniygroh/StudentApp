# Інструкція з використання системи НаУКМА

## 1. Запуск програми
Система підтримує два режими роботи:

* **Локальний режим:** Запустіть файл `Main.java`. Програма автоматично підвантажить існуючі бази даних з папки `files/` (або створить нові).
* **Мережевий режим:** 1. Запустіть `TcpServer.java` (порт 8080).
    2. Запустіть `TcpClient.java` для відправки текстових запитів (наприклад, `GET_STUDENT 1`).

## 2. Авторизація та Реєстрація
* **Логін:** Введіть email та пароль.
* **Реєстрація:** Пароль має містити >= 8 символів, 1 велику літеру, 1 малу літеру та 1 цифру. 
* **Унікальність:** Система не дозволяє реєстрацію дублікатів email-адрес.

## 3. Ролі користувачів
*  **USER:** Тільки читання (пошук, списки, звіти).
*  **MANAGER:** CRUD-операції (створення, оновлення, видалення) в реєстрах.
*  **ADMIN:** Повні права + "Меню користувачів" (блокування, зміна ролей/паролів).

## 4. Навігація в Головному меню
Основні реєстри доступні за номерами:
1. `Student's registry` — Студенти.
2. `Faculty's registry` — Факультети.
3. `Teacher's registry` — Викладачі.
4. `Department's registry` — Кафедри.
5. `User's registry` — Тільки для ADMIN.

## 5. Основні операції (CRUD)
* **Додавання (Add):** Покрокове заповнення даних з валідацією.
* **Редагування (Update):** Вибір запису за ID та вибір конкретного поля для зміни.
* **Пошук (Get/Show):** Пошук за ID, ім'ям або фільтрація за факультетом/курсом.
* **Звіти (Reports):** Аналітика через Stream API (середній вік, розподіл за формами навчання).

## 6. Завершення роботи
Використовуйте `0 - leave` для виходу. Дані зберігаються автоматично у зашифрованих `.ser` файлах (алгоритм XOR).
